# دليل إعداد متجر وائل شداد على Supabase + Netlify

اتبع الخطوات بالترتيب. كلها مجانية ولا تحتاج بطاقة ائتمان.

---

## 1) إنشاء مشروع Supabase
1. ادخل على <https://supabase.com> واعمل حساب (أو سجّل دخول).
2. اضغط **New Project**.
3. اختر اسم (مثلاً `wael-shaddad`)، وكلمة مرور قوية لقاعدة البيانات (احفظها)، والمنطقة الأقرب (Frankfurt مناسبة لمصر).
4. استنى دقيقة لحد ما المشروع يجهز.

## 2) تشغيل ملف قاعدة البيانات
1. من القائمة الجانبية اختر **SQL Editor** > **New query**.
2. افتح ملف [`db/schema.sql`](db/schema.sql) من المشروع، انسخ **كل محتواه**، والصقه في المحرر.
3. اضغط **Run** (أو Ctrl+Enter).
4. المفروض تظهر رسالة نجاح، ويتعمل: جدول `products`، جدول `orders`، سياسات الأمان، bucket الصور، و7 منتجات تجريبية.

## 3) التأكد من Storage
1. اختر **Storage** من القائمة الجانبية.
2. لازم تلاقي bucket اسمه **`product-images`** وحالته **Public**.
   - لو مش موجود لأي سبب: اعمله يدويًا (**New bucket** > الاسم `product-images` > فعّل **Public bucket**).

## 4) إنشاء حساب الأدمن
1. اختر **Authentication** > **Users** > **Add user** > **Create new user**.
2. اكتب إيميل وكلمة مرور للأدمن (دي بيانات دخولك للوحة التحكم `admin/login.html`).
3. فعّل **Auto Confirm User** عشان تقدر تسجّل دخول على طول.

## 5) نسخ مفاتيح الاتصال للموقع
1. اختر **Project Settings** (الترس) > **API**.
2. انسخ:
   - **Project URL**
   - **anon public** key
3. افتح ملف [`js/supabase.js`](js/supabase.js) في المشروع، واستبدل:
   ```js
   const SUPABASE_URL  = 'ضع_الـ_Project_URL_هنا';
   const SUPABASE_ANON = 'ضع_الـ_anon_key_هنا';
   ```

## 6) تشغيل الموقع محليًا للتجربة
الموقع بيستخدم `fetch`، فلازم يشتغل عبر سيرفر (مش فتح الملف مباشرة):
```bash
cd wail_furniture
python -m http.server 8000
```
افتح <http://localhost:8000> — المفروض تشوف المنتجات التجريبية.
> للتأكد من الاتصال: افتح Console في المتصفح، هتلاقي رسالة `✅ Supabase connected`.

## 7) النشر على Netlify
1. ادخل <https://netlify.com> > **Add new site** > **Deploy manually**.
2. اسحب مجلد المشروع كله (`wail_furniture`) على الصفحة.
3. هيطلعلك رابط الموقع مباشرة. (مفيش build step — كله static.)
   - بديل: اربط المشروع بـ GitHub repo للنشر التلقائي مع كل push.

## ملاحظات
- **بدون أسعار:** الموقع لا يعرض أي أسعار للعميل. حقل `price` في لوحة التحكم اختياري وداخلي لك فقط.
- **الطلبات:** بتتسجّل في جدول `orders` + بتفتح رسالة واتساب جاهزة على الرقم `+201272073305`.
- **رفع صور المنتجات:** من لوحة التحكم (`admin/dashboard.html`) — بتترفع تلقائيًا على bucket `product-images`.
- لو الصفحات اشتغلت من غير ما تحط المفاتيح، هتلاقي منتجات تجريبية من `data/products.json` (وضع احتياطي للتجربة فقط).
